{"markdown":"[Jack Henschel's Blog](https://blog.cubieserver.de/)\n\n=====================================================\n\n[](https://blog.cubieserver.de/)\n[](mailto:jack.henschel@mailbox.org)\n[](https://github.com/jacksgt)\n[](https://linkedin.com/in/jack-henschel)\n[](https://social.secclo.community/jack)\n\n [Home](https://blog.cubieserver.de/)\n\nRunning banking apps and other suspects on Custom ROMs with root\n================================================================\n\nPosted on Jan 13, 2025\n\n[#android](https://blog.cubieserver.de/tags/android/)\n [#lineageos](https://blog.cubieserver.de/tags/lineageos/)\n\nThere are a number of reasons why it can be a good idea to install an alternative operating system (aka. _Custom ROM_) on an Android device: better performance, more privacy, longer security updates and of course customization. Whatever the reason may be, users of custom ROMs (including myself) usually struggle with the fact that many banking apps and other “high security” apps refuse to run on rooted devices.\n\nIn this post I will share how I managed to work around this issue. We’ll go over [installing LineageOS with Google Apps](https://blog.cubieserver.de/2025/running-banking-apps-and-other-suspects-on-custom-roms-with-root/#install-custom-rom-with-gapps)\n, [installing Magisk](https://blog.cubieserver.de/2025/running-banking-apps-and-other-suspects-on-custom-roms-with-root/#install-magisk)\n and [PlayIntegrityFix module](https://blog.cubieserver.de/2025/running-banking-apps-and-other-suspects-on-custom-roms-with-root/#install-playintegrityfix)\n, configuring [Magisk DenyList to hide root](https://blog.cubieserver.de/2025/running-banking-apps-and-other-suspects-on-custom-roms-with-root/#hide-magisk)\n and finally testing the setup with the [Play Integrity API Checker](https://blog.cubieserver.de/2025/running-banking-apps-and-other-suspects-on-custom-roms-with-root/#testing)\n. For reference, I’m using LineageOS 20 (Android 13) on a Motorola Moto G7 Plus (`lake`). I got the apps `DKB`, `Revolut`, `SAP Concur` as well as `Google Wallet` working with this approach.\n\n[\\#](https://blog.cubieserver.de/2025/running-banking-apps-and-other-suspects-on-custom-roms-with-root/#install-custom-rom-with-gapps)\n  Install Custom ROM with GApps\n----------------------------------------------------------------------------------------------------------------------------------------------------------------------\n\nThe first step will be installing the custom ROM (in this case LinageOS) along with the necessary components: GApps and Magisk. Installing a custom ROM requires unlocking the bootloader of your phone first, of course. Since procedure is slightly different for each phone, I won’t repeat it here, but the LineageOS wiki has excellent instructions. For example you can find the instructions for the Moto G7 Plus here: [https://wiki.lineageos.org/devices/lake/install/#unlocking-the-bootloader](https://wiki.lineageos.org/devices/lake/install/#unlocking-the-bootloader)\n.\n\nOut of the box LineageOS does not come with Google Play Services since this is a proprietary component of Android. However, we’ll need this component for the Google Wallet integration and other security related features. Therefore, download the _Google Apps_ package from the LineageOS site: [https://wiki.lineageos.org/devices/lake/install/#unlocking-the-bootloader](https://wiki.lineageos.org/devices/lake/install/#unlocking-the-bootloader)\n - note that you’ll a version matching the ROM version number.\n\nJust like the ROM itself, the ZIP file is installed via ADB sideloading. **Make sure to run this step immediately after installing the ROM.**\n\n> **Note**: for all of the following procedures (`adb` / `fastboot`) I had to use a USB2 port since it would not work with a USB3 port. You can also use a USB hub, since these usually downgrade v3 to v2. See: [https://reddit.com/r/LineageOS/comments/gofxb6/fastboot\\_oem\\_get\\_unlock\\_data\\_outputs\\_nothing/](https://reddit.com/r/LineageOS/comments/gofxb6/fastboot_oem_get_unlock_data_outputs_nothing/)\n\nAfter unlocking the bootloader, the installation procedure is as follows:\n\n*   Check the devices is visible: `adb devices -l`\n*   Reboot into recovery mode: `adb reboot bootloader`\n*   Flash Lineage recovery `fastboot flash boot lineage-20.0-20230110-recovery-lake.img`\n*   Power off phone and reboot into recovery\n*   Enable ADB in recovery mode (in the “Advanced” section) and perform factory reset\n*   Apply update > Apply from ADB\n*   Run `adb sideload lineage-20.0-20230110-nightly-lake-signed.zip` and wait until it completes\n*   Advanced > Reboot to Recovery\n*   Apply update > Apply from ADB\n*   Run `adb sideload MindTheGapps-13.0.0-arm64-20221025_100653.zip` (confirm installation without signature) - make sure the last line is “Install completed with status 0”\n*   Finally click on _Reboot system now_\n\nThe phone should successfully boot into LineageOS and you can start setting up the phone.\n\n![Installing LineageOS on Moto G7 Plus with an old USB hub to force USB2.0](https://blog.cubieserver.de/2025/running-banking-apps-and-other-suspects-on-custom-roms-with-root/install-lineage.jpeg)\n\n[\\#](https://blog.cubieserver.de/2025/running-banking-apps-and-other-suspects-on-custom-roms-with-root/#install-magisk)\n  Install Magisk\n----------------------------------------------------------------------------------------------------------------------------------------\n\nAt this point we need to install Magisk, which provides “systemless” root access. Systemless means that we don’t need to change any data on the Android partition. Instead Magisk works by injecting special code early in the boot process of the device. The project has a [comprehensive installation guide](https://topjohnwu.github.io/Magisk/install.html)\n, so make sure to read it all and apply the appropriate procedure for your phone.\n\nIn a nutshell: the `boot.img` of LineageOS we installed in the previous step needs to be modified with the Magisk code. The [Magisk app](https://github.com/topjohnwu/Magisk/releases)\n can do this automatically for us, we just need to provide it the correct file.\n\nUsually you can simply download the `boot.img` from the LineageOS website (make sure to download the version that matches the LineageOS ZIP!). In my case the `boot.img` was unfortunately not available, therefore I had to use the following workaround: extracting the boot image from the LineageOS ZIP itself. I used the [payload-dumper-go](https://github.com/ssut/payload-dumper-go)\n project for this task:\n\n|     |     |\n| --- | --- |\n| 1<br>    2<br>    3<br>    4<br>    5 | unzip lineageos-20.0-*.zip<br>    cd lineageos<br>    go install github.com/ssut/payload-dumper-go@latest<br>    ~/go/bin/payload-dumper-go payload.bin<br>    adb push extracted/boot.img /sdcard/Download/boot.img |\n\nNow open the Magisk app, click on “Install” (top-right corner) and patch the image by checking the “Recovery Mode” option and choosing a the `boot.img` file we just copied via “Select and Patch a File” ([see Magisk instructions](https://topjohnwu.github.io/Magisk/install.html)\n).\n\n![Magisk App](https://blog.cubieserver.de/2025/running-banking-apps-and-other-suspects-on-custom-roms-with-root/magisk.png)\n\nAfterwards we can copy the patched file back to the PC:\n\n|     |     |\n| --- | --- |\n| 1   | adb pull /sdcard/Download/magisk_patched_[random_strings].img . |\n\nReboot into the recovery mode, then flash the new (patched )recovery image:\n\n|     |     |\n| --- | --- |\n| 1<br>    2<br>    3<br>    4<br>    5<br>    6<br>    7 | adb reboot bootloader<br>    <br>    fastboot devices<br>    <br>    fastboot flash boot magisk_patched-27000_0P3ep.img -v<br>    <br>    fastboot reboot |\n\n> Note: if the fastboot process seems stuck while sending, try the following: unplug the phone from the PC before starting, run the `fastboot flash ...` command, wait for the `> waiting for device` message to appear, then plug in the phone. Reference: [https://unix.stackexchange.com/a/732659](https://unix.stackexchange.com/a/732659)\n\nNow we can reboot the phone once more. It should start up successfully and now has Magisk capabilities (you can verify this by opening the Magisk app, it should say `Installed <VERSION>`).\n\n[\\#](https://blog.cubieserver.de/2025/running-banking-apps-and-other-suspects-on-custom-roms-with-root/#install-playintegrityfix)\n  Install PlayIntegrityFix\n------------------------------------------------------------------------------------------------------------------------------------------------------------\n\n[PlayIntegrityFix](https://github.com/chiteroman/PlayIntegrityFix/releases)\n is a Magisk module that hooks into the _Play Integrity_ and _SafetyNet_ frameworks (provided by Google Play Services) and make them return accepted values.\n\nOn the phone, download the latest ZIP from the [releases page](https://github.com/chiteroman/PlayIntegrityFix/releases)\n and install via the “Modules” tab at the bottom of the Magisk App, then reboot once again.\n\n[\\#](https://blog.cubieserver.de/2025/running-banking-apps-and-other-suspects-on-custom-roms-with-root/#testing)\n  Testing\n--------------------------------------------------------------------------------------------------------------------------\n\nAt this point we need to check the basic functionality, for which we can use a special app called [Play Integrity API Checker](https://github.com/1nikolas/play-integrity-checker-app)\n that should be installed from the [Google Play Store](https://play.google.com/store/apps/details?id=gr.nikolasspyr.integritycheck)\n.\n\nThis should return:\n\n![Result of Play Integrity API checker app](https://blog.cubieserver.de/2025/running-banking-apps-and-other-suspects-on-custom-roms-with-root/play-integrity-api-checker.png)\n\n|     |     |\n| --- | --- |\n| 1<br>    2<br>    3 | MEETS_DEVICE_INTEGRITY: OK<br>    MEETS_BASIC_INTEGRITY: OK<br>    MEETS_STRONG_INTEGRITY: FAILED |\n\nDue to the fact that the bootloader is unlocked, we can’t pass the `MEETS_STRONG_INTEGRITY` check, but that’s not necessary for most use cases. The important ones are the `MEETS_DEVICE_INTEGRITY` and `MEETS_BASIC_INTEGRITY` checks.\n\nAt this point, most financial apps as well as Google Wallet should be working.\n\n[\\#](https://blog.cubieserver.de/2025/running-banking-apps-and-other-suspects-on-custom-roms-with-root/#hide-magisk)\n  Hide Magisk\n----------------------------------------------------------------------------------------------------------------------------------\n\nSome apps implement additional, custom checks for the device they run on. To trick these apps into working, we need to “hide” the fact that the phone has root capabilties. Thanks to Magisk’s powerful capabilities, this is possible with the built-in _DenyList_ feature. Open up the Magisk app, click on the Settings icon in the top-right corner, scroll down and enable the _Zygisk_ and _Enforce DenyList_ options.\n\n![Activate Magisk DenyList](https://blog.cubieserver.de/2025/running-banking-apps-and-other-suspects-on-custom-roms-with-root/magisk-settings.png)\n\nThen configure the _DenyList_ as follows:\n\n*   check _Android Services Library_\n*   check any other app that you want to hide root capabilities from (in my case DKB)\n\n![Configuring Magisk App DenyList](https://blog.cubieserver.de/2025/running-banking-apps-and-other-suspects-on-custom-roms-with-root/magisk-denylist.png)\n\nIf this is still not enough, you may also want to try to rename the Magisk app (see Settings > _MagiskHide_).\n\nFinally, it’s a good idea to clear the local storage of the app we are trying to hide from. This ensures that the previous “bad” state is not persisted across restarts. Now the app should work even though we a custom ROM with root!\n\nGood luck!\n\n![Author Avatar](https://blog.cubieserver.de/images/wheel.jpg)\n\nAuthor:   [Jack
