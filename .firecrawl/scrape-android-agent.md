No app installed. No footprint. Pure automation.\n\nDefine **skills** for any app, run them from the dashboard or API, scale across a phone farm.\n\n**The ghost taps what you'd tap**\n\n*   50+ ADB methods — tap, swipe, type, clipboard, stealth variants\n*   Live phone screen streaming (MJPEG and WebRTC)\n*   Interactive touch-to-tap on the streamed screen\n*   Multi-device phone farm with per-device job queues\n\n**Forge reusable skills for any app**\n\n*   YAML-based UI element definitions per app\n*   Python action classes with precondition checks\n*   Multi-step workflows that chain actions together\n*   Built-in skills for TikTok and Play Store\n*   **Skill Hub** — browse, search, and install skills from the community registry\n*   Install from CLI: `android-agent skill install tiktok`\n\n**Teach the ghost new tricks**\n\n*   BFS-based auto app explorer — discovers every screen and transition\n*   LLM-assisted Skill Creator — chat with AI while viewing the live device stream\n*   The AI identifies UI elements and generates action/workflow code\n\n**Scale the haunting**\n\n*   Multi-device phone farm with per-device job queues\n*   Bot runner: queue, schedule, and monitor automation jobs\n*   Per-device integration tests with ADB screen recording\n\n* * *\n\nRequirements\n------------\n\n[](https://github.com/ghost-in-the-droid/android-agent#requirements)\n\n*   **Python 3.10+**\n*   **Android phone** with USB debugging enabled (Settings > Developer Options > USB Debugging)\n*   **ADB** installed and on PATH (`adb devices` should list your phone)\n*   **Node.js 18+** (for the frontend dev server)\n\n* * *\n\nQuick Start\n-----------\n\n[](https://github.com/ghost-in-the-droid/android-agent#quick-start)\n\n```shell\n# 1. Vite proxies API calls to the backend during development.\n3.  **Skills** define how to interact with a specific app. Each skill has a `elements.yaml` (UI elements), Python actions (atomic operations), and workflows (multi-step sequences).\n4.  **Bots** are long-running subprocess scripts that use the `Device` class from `bots/common/adb.py` to control the phone.
